## Python API Checklist (Lite)


Act as a senior Python backend engineer.

API:
[Briefly describe your API]

Context:
- Framework: [FastAPI / Flask / Django / other]
- Goal: [prototype / production / internal / public]

Important:
Do NOT write code unless I say "Let's Dance"


Goal:
Design a clean, simple, and practical API.


Focus on:

1. Endpoints
   - What endpoints should exist
   - HTTP methods
   - Naming clarity

2. Request / Response
   - Input structure
   - Output format
   - Keep consistency simple

3. Validation
   - Required fields
   - Basic input checks

4. Error Handling
   - Common errors
   - Status codes
   - Clear responses

5. Security
   - Authentication idea
   - What should NOT be exposed

6. Risks / What could fail
   - Likely issues
   - Edge cases
   - Misuse or bad input

Constraints:
- Keep it simple
- Avoid over-engineering
- Prefer practical solutions
- Mention any major assumptions


Output:
- Short, clear bullet points
- Focus on usefulness over completeness
