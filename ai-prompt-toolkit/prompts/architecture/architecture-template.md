Architecture Prompt Template 
Design systems with clear and practical decisions, to prevent over-engineering. 

Template: 

Act as a software architect.

System:
[Describe system]

Constraints:
- Scale: [small / medium / large]
- Time: [tight / moderate / long-term]
- Team: [solo / small / large]
- Priority: [speed / scalability / maintainability]

Important Rule:
Do NOT write any code unless I explicitly say: "Let's Dance"

Mental Model:
- What → system architecture
- How → structured reasoning with tradeoffs
- Why → real-world viability
- Scope → MVP-first design
- What NOT to do → avoid premature complexity

Constraints:
- Prefer simple solutions first
- Avoid unnecessary abstraction
- Focus on practical implementation
- Highlight tradeoffs clearly
- Identify risks and failure points

Resource Rule:
- Include 2–3 high-value links only

Please provide:

1. High-level architecture overview
2. Core components and responsibilities
3. Data flow explanation
4. Technology choices (with reasoning)
5. Key tradeoffs
6. Simplest viable architecture (MVP)
7. Evolution path (how it scales)
8. Risks and failure points
9. Performance considerations
10. What should NOT be built yet

Output requirements:
- Clear sections
- Bullet points
- Practical, real-world reasoning

If a recommendation introduces complexity, explicitly justify why it is necessary.
