Testing Checklist Prompt
*Quickly evaluate app quality before submission*

Template 

Act as a QA Engineer. 

Application: 
[Describe App]

Please provide a practical, prioritized pre-submission review checklist.

Output requirements:
- Use the 10 sections below
- For each section, include:
  - Priority: High / Medium / Low
  - A short explanation of what to check
  - 3 to 7 practical checklist items
- Focus on issues most likely to affect code review quality
- Emphasize problems a reviewer would notice quickly
- Keep the recommendations concise, actionable, and time-efficient
- Avoid exhaustive low-value checks

Sections:
1. Functional testing
2. Edge cases
3. Error handling
4. UI / UX checks
5. Performance checks
6. API checks (if applicable)
7. Security basics
8. Code quality review
9. Reviewer focus areas
10. Quick fixes under 15 minutes
